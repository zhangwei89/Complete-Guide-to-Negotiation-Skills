# 附录 A 谈判准备 Checklist

## 使用说明

这份 Checklist 适用于商务谈判、职场谈判、销售谈判、采购谈判等任何正式或半正式的谈判场景。在谈判开始前 7 天,逐项核对,确保准备充分。

## 一、信息收集表

### 1. 对方基本信息

- 公司名称、注册地、规模。
- 主要业务、行业地位、市场份额。
- 关键决策人(职位、姓名、背景、风格)。
- 决策链结构(谁有最终决策权)。
- 内部协作模式(谁影响谁)。

### 2. 对方近期动态

- 最近的新闻报道、媒体报道。
- 最近的财报或行业报告。
- 最近的招聘动向(扩张 or 收缩)。
- 最近的资本运作(融资、并购、上市)。
- 最近的战略动作(新产品、新市场)。

### 3. 对方历史合作

- 过去合作的客户名单。
- 过去合作的项目案例。
- 过去合作中出现的常见问题。
- 过去的合同关键条款。

### 4. 对方关键诉求

- 对方表面诉求(对方明确表达)。
- 对方真实诉求(对方未明确表达,通过调研推断)。
- 对方底线(对方能接受的最低条件)。
- 对方痛点(对方最想解决的问题)。
- 对方备选方案(对方可能的替代方案)。

### 5. 对方关键弱点

- 对方的资金压力。
- 对方的时间压力。
- 对方的关系压力。
- 对方的认知盲区。
- 对方的情绪触发器。

## 二、利益分析表

### 1. 对方利益矩阵

- 核心利益(必须满足)。
- 重要利益(尽量满足)。
- 一般利益(可以谈判)。
- 次要利益(可以放弃)。

### 2. 我方利益矩阵

- 核心利益(必须守住)。
- 重要利益(尽量争取)。
- 一般利益(可以谈判)。
- 次要利益(可以放弃)。

### 3. 共同利益点

- 双方都希望达成的目标。
- 双方都希望避免的后果。
- 双方都能接受的中间地带。

### 4. 冲突利益点

- 双方目标对立的部分。
- 冲突的严重程度。
- 是否有创造性解决方案的可能。

## 三、BATNA 分析表

### 1. 我方 BATNA

- 如果本次谈判失败,我方的最佳替代方案是什么?
- 这个替代方案的价值是多少?
- 这个替代方案的实施成本是多少?
- 这个替代方案的时间周期是多少?
- 实施替代方案的风险是什么?

### 2. 对方 BATNA

- 如果本次谈判失败,对方的最佳替代方案是什么?
- 这个替代方案的价值是多少?
- 对方实施替代方案的难度?
- 对方实施替代方案的时间周期?
- 对方实施替代方案的偏好度?

### 3. BATNA 对比分析

- 双方 BATNA 强弱对比。
- 谁更可能先退出谈判。
- 谁拥有 BATNA 优势。
- 如何用 BATNA 增强谈判力。

## 四、底线与目标区间

### 1. 价格底线

- 我方的最高价格(如果购买方)。
- 我方的最低价格(如果销售方)。
- 价格的让步空间(可让步的比例)。
- 价格让步的次数限制。
- 价格让步的触发条件。

### 2. 关键条款底线

- 付款方式底线。
- 交期底线。
- 质量标准底线。
- 违约责任底线。
- 知识产权底线。
- 争议解决底线。

### 3. 目标三档

- 理想结果(超额完成)。
- 期望结果(达成主要目标)。
- 底线结果(可接受的最低条件)。

### 4. 退出条件

- 对方提出的什么条件会让我方必须退出?
- 退出的具体触发条件是什么?
- 退出的代价是什么?
- 退出的备选方案是什么?

## 五、风险评估表

### 1. 战略风险

- 谈判失败对公司业务的影响。
- 谈判失败对客户关系的影响。
- 谈判失败对市场声誉的影响。

### 2. 财务风险

- 谈判失败的直接经济损失。
- 谈判失败的间接经济损失。
- 谈判结果的不确定性幅度。

### 3. 法律风险

- 合同条款的法律风险。
- 适用法律的不确定性。
- 争议解决的成本。

### 4. 关系风险

- 谈判失败对客户关系的影响。
- 谈判失败对行业口碑的影响。
- 谈判失败对未来合作的影响。

### 5. 执行风险

- 合同签订后的执行风险。
- 对方的履约能力不确定性。
- 我方的履约能力评估。

## 六、谈判策略表

### 1. 整体策略

- 我方采用什么谈判哲学(零和 / 共赢 / 条件)?
- 我方采用什么节奏策略(快速推进 / 慢速推进 / 节奏交替)?
- 我方采用什么让步策略(高开低走 / 等价交换 / 限时让步)?

### 2. 开局策略

- 第一印象如何塑造?
- 如何建立合作氛围?
- 如何表达我方诉求?
- 如何了解对方诉求?

### 3. 中期策略

- 如何应对对方的极端要求?
- 如何使用信息优势?
- 如何引入第三方 / 权威?
- 如何处理僵局?

### 4. 收尾策略

- 如何识别对方让步信号?
- 如何把握成交时机?
- 如何处理最后的分歧?
- 如何确认书面协议?

### 5. 备选策略

- 如果 A 计划失败,预备 B 计划。
- 如果对方提出意外要求,预备应对。
- 如果出现僵局,预备退出策略。

## 七、团队准备表

### 1. 角色分配

- 主谈(谁主谈价格、商务条款)。
- 技术专家(谁解答技术问题)。
- 商务经理(谁管合同条款)。
- 记录员(谁做会议记录)。
- 后备成员(谁提供支持)。

### 2. 团队对齐

- 团队成员对目标的理解一致。
- 团队成员对底线的理解一致。
- 团队成员对口径的理解一致。
- 团队成员对角色的理解一致。

### 3. 应急预案

- 对方情绪失控的应对。
- 团队内部出现分歧的应对。
- 议题出现意外的应对。
- 信息出现误差的应对。

## 八、资料准备表

### 1. 我方资料

- 公司介绍(精简版)。
- 产品 / 方案手册。
- 报价单(最新版)。
- 合同模板。
- 案例集(2 到 3 个)。
- 公司资质证书。

### 2. 对方资料

- 对方公司档案。
- 对方关键人物画像。
- 对方历史合作记录。
- 对方行业研究报告。

### 3. 谈判工具

- 笔记本电脑 + 备用设备。
- 投影仪 / 投屏设备。
- 名片。
- 文具(笔记本、笔)。
- 备用资料(打印版)。

## 九、谈判前最后检查(谈判开始前 24 小时)

- 团队对齐会议已开(目标、分工、底线)。
- 模拟演练已完成(至少 1 次)。
- 资料准备齐全(资料包 + 备用)。
- 心理准备到位(精神、睡眠、饮食)。
- 物理准备到位(衣着、装备、行程)。
- 后勤安排到位(场地、餐饮、设备)。
- 应急联系到位(内部、外部)。
- 心态调整到位(好奇、准备、专业)。

## 十、谈判开始前 15 分钟最后检查

- 团队成员到齐,状态正常。
- 资料、设备齐全。
- 心态平静、专注。
- 身体状态良好(无饥饿、无疲惫)。
- 对方状态观察(对方团队构成、气氛)。

## 附录使用建议

1. 每次重要谈判前 7 天,完整走一遍这份 Checklist。
2. 用纸质或电子版本记录,留档。
3. 谈判后做复盘时,对照 Checklist 检查哪些环节做好了、哪些环节需要改进。
4. 每季度更新一次 Checklist,根据经验沉淀改进项。
5. 团队内部共享 Checklist,让所有成员都有统一的准备标准。

---

这份 Checklist 不是「死规定」,而是「活的工具」。随着你的经验增长,你会不断往里面添加新内容、删除冗余项、调整优先级。把 Checklist 当作你的「谈判前必备清单」,你会发现准备时间大幅减少,准备质量大幅提升。
