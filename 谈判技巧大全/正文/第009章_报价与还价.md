# 第 9 章 报价与还价

## 开篇：林涵少赚 80 万的那次报价

林涵第一次独立谈一个 380 万的项目，她经历了 4 轮谈判。

第 1 轮：她开口报 380 万。客户皱眉。她心里慌了一下。

第 2 轮：客户还价 320 万。她心一软："340 万吧。"客户不说话。

第 3 轮：客户还价 300 万。她沉默 5 秒："300 万也行。"

第 4 轮：客户说："再加 50 万的增值服务（之前没提）。"

最后客户同意 350 万成交。

林涵后来用一周时间算账，她本可以报 420 万，留出 40 万的让步空间，最终成交 380 万。

她多让了 30 万，加上"客户要 50 万增值服务"，实际损失 80 万。

她后来写下："我最大的错误不是报价太高，而是我没有设计报价策略。"

这一章我们就来设计报价策略。

---

## 一、高开低走 vs 低开高走：两种报价策略

### 策略 1：高开低走（先高后低）

先报一个偏高的价格，然后逐步让步。

优势：

- 留出让步空间。
- 让对方感觉"他让步了"。
- 让你最终的成交价落在理想区间。

劣势：

- 对方可能直接拒绝。
- 你可能因为"开得太高"失去信誉。
- 你需要花更多时间在让步博弈上。

适用场景：

- 你有信息优势。
- 对方是新手。
- 你处于强势地位。

### 策略 2：低开高走（先低后高）

先报一个偏低的价格，然后逐步加价。

优势：

- 对方容易接受。
- 快速推进谈判。
- 你不需要花太多时间在让步博弈上。

劣势：

- 你没有让步空间。
- 加价会被对方警惕。
- 你最终可能落在底线以下。

适用场景：

- 你处于弱势地位。
- 对方是高手。
- 你需要这单维持运营。

### 决策规则：什么时候用哪个？

| 你的情境 | 高开低走 | 低开高走 |
|---------|---------|---------|
| 你有信息优势 | 是 | |
| 你是大公司 | 是 | |
| 对方是新手 | 是 | |
| 你处于弱势地位 | | 是 |
| 你需要这单 | | 是 |
| 对方是高手 | | 是 |
| 你是小公司 / 新公司 | | 是 |

实战建议：

- 大多数谈判高手默认用"高开低走"。
- 但如果你发现对方很强势 / 你很需要这单，立刻切换到"低开高走"。

## 二、锚定效应：第一报价的 4 个设置

锚定效应（anchoring effect）：第一报价成为后续所有判断的参考系。

### 第一报价的 4 个设置

设置 1：理想值 + 10-20%

报价 = 你的理想值 + 10-20%。

示例：你的理想值是 380 万，你报价 420-460 万。

设置 2：行业标准 + 20-30%

报价 = 行业标准 + 20-30%。

示例：行业标准 300 万，你报价 360-390 万。

设置 3：成本 + 50-100%

报价 = 你的成本 + 50-100%。

示例：你的成本 200 万，你报价 300-400 万。

设置 4：竞品均价 + 20-30%

报价 = 竞品均价 + 20-30%。

示例：竞品均价 320 万，你报价 380-420 万。

### 第一报价的 3 个规则

规则 1：具体

错误："大概 380 万左右"，对方立刻往下压。

正确："我们的报价是 380 万 8000 元整。"

具体数字（包括小数点）让对方感觉"这是精确计算的"。

规则 2：留余地

你的报价应该比你的理想值高 10-20%。这给你留下让步空间。

规则 3：基于事实

你的报价应该有理由。

实战话术：

- "基于您的规模、行业、和我们的成本结构，我们的报价是 380 万。"
- "按照行业惯例和您的具体需求，报价 380 万是合理的。"

---

## 三、价格让步的 4 条铁律

让步不是"示弱"，让步是"策略"。让错了，比不让更糟。

### 铁律 1：不要等量让步

错误：

- 你让 40 万，对方让 40 万。
- 你让 30 万，对方让 30 万。

这种"等量让步"让你没有优势。

正确：

- 你让 20 万，对方让 40 万。
- 你让 10 万，对方让 30 万。

让步比例应该是：每次你的让步幅度递减（10% → 7% → 5% → 3%），对方的让步幅度也要递减，但**总让步幅度你比对方大**。

### 铁律 2：不要连续让步

错误：

- 连续 3 轮你都让步。
- 每次让步都比上次大。

对方会知道"他只要继续压，我就会继续让"。

正确：

- 每次让步之间至少间隔一轮"非让步"（你给信息 / 提问题 / 加项）。
- 让步幅度越来越小。

### 铁律 3：不要无理由让步

错误：

- 客户没说任何理由，你直接让步。

正确：

- 客户给你理由（订单量 / 长期合作 / 现款 / 转介绍），你让步。
- 客户没给你理由，你找理由（让他先让步）。

### 铁律 4：让步必换

错误：

- 你让步 10%，对方没给你任何东西。

正确：

- 你让步 10%，对方让步 5% + 给你介绍 1 个客户 + 给你写 1 个案例。

让步必换，每次让步都换回价值。

## 四、多方案报价：good-better-best 三档设计

林涵那个 380 万失败后，她学了一个最关键的报价技巧，多方案报价。

### 三档设计的核心

不要只给一个价格，给 3 个价格，让对方"选择"。

三档的结构：

- Good 档（基础）：能解决问题，但服务 / 时间 / 范围有限。
- Better 档（推荐）：能解决问题，服务 / 时间 / 范围中等。
- Best 档（全面）：能解决问题 + 加项 + 长期价值。

### 一个示例（林涵后来设计的方案）

林涵后来的方案：

| 档位 | 价格 | 包含内容 | 适合客户 |
|------|------|---------|---------|
| Good 档 | 280 万 | 基础 ERP + 6 个月运维 | 短期试点 |
| Better 档 | 380 万 | 完整 ERP + 12 个月运维 + 培训 | 主流客户 |
| Best 档 | 480 万 | 完整 ERP + 24 个月运维 + 培训 + VIP 客户画像建模 | 长期战略合作 |

### 三档报价的 3 个好处

好处 1：让对方"选择"而不是"拒绝"。

对方不再说"380 万太贵"，而是说"我选 Better 档"或"我选 Good 档"。

好处 2：锚定效应自动生效。

客户看到 480 万的 Best 档，会觉得 380 万的 Better 档是"中间选项"，这正是你要的。

好处 3：对方有"降档"的心理出路。

如果客户的预算只有 350 万，他不会离开，他会选 Good 档，你保住了客户。

### 三档设计的 3 个原则

原则 1：3 档之间有清晰差距

不要"380 / 390 / 400"，差距太小，对方感觉没区别。

要"280 / 380 / 480"，差距明显，对方必须做选择。

原则 2：中间档是"推荐档"

把最好的"性价比"放在 Better 档，你的报价默认对方会选中间档。

原则 3：把客户引导到 Better 档

话术：

- "我建议您考虑 Better 档，这是大多数客户的选择。"
- "Good 档功能有限，Best 档对您来说可能有些功能用不上。"

## 五、条件交换：让价换让价 / 让价换时间 / 让价换规模

让步的本质是"交换"。每一次让步都要换回东西。

### 交换类型 1：让价换让价

你让步 10%，对方让步 5% + 别的东西。

实战话术：

- "我可以让步 10%，但希望您也能在付款方式上让步。"
- "我可以让步 5%，但希望您能签 2 年期合同。"

### 交换类型 2：让价换时间

你让步 5%，对方提前付款 / 长期合作。

实战话术：

- "我可以让步 5%，但希望您能在签合同 1 周内付首款。"
- "我可以让步 10%，但希望您能签 3 年期合同。"

### 交换类型 3：让价换规模

你让步 5%，对方承诺更大的订单量。

实战话术：

- "我可以让步 5%，但希望您能承诺年订单量不低于 500 万。"
- "我可以让步 10%，但希望您能介绍 2 个类似客户给我们。"

### 交换类型 4：让价换案例

你让步 5%，对方同意做案例 / 允许你引用。

实战话术：

- "我可以让步 5%，但希望您能在上线后允许我们把您作为案例分享。"

### 交换类型 5：让价换关系

你让步 5%，对方承诺更高的合作深度。

实战话术：

- "我可以让步 5%，但希望您能优先选择我们作为下一个项目的合作伙伴。"

实战原则：

- 交换的"价值"要相等，你让步 5%，对方让步的也应该是 5% 左右的价值。
- 不要让步太大，每次让步幅度递减（10% → 7% → 5% → 3%）。
- 不要在压力下让步，只在你主动选择让步时让步。

## 六、应对压价：4 招

### 招数 1：量化价值

"我们贵，但贵在哪？"

错误：

- "我们这个方案好"，空洞。

正确：

- "我们这个方案能让您每月节省 8 万元运营成本，3 年下来累计 288 万。这比您给我们多付的 60 万高 4 倍。"

### 招数 2：加项

"价格可以不动，但我们可以加项。"

错误：

- "我们不能再降了"，硬顶。

正确：

- "价格我们这边有压力。但我们可以给您加一项免费的 VIP 服务，您看如何？"

### 招数 3：切分

"价格不动，我们切分服务。"

错误：

- "价格不能再降了"，硬顶。

正确：

- "您可以只买我们方案的 50%，这样价格能减半。但另外 50% 您可以在 6 个月后再决定。"

### 招数 4：退场

"如果价格谈不拢，我们也没关系。"

错误：

- 假装要退场（虚张声势），对方识破。

正确：

- 真的准备退场。"如果价格实在谈不拢，我们也很遗憾，但您可以考虑其他方案。"

实战建议：

- 4 招不要都用，选 1-2 招。
- 招数 1（量化价值）是最稳的，永远先用。
- 招数 4（退场）只能在底线明确时用，没准备好别用。

---

## 七、应对涨价：3 招

### 招数 1：核查成本

"你涨价的具体原因是什么？"

对方说："我们要涨价 10%。"你问："您能详细说说涨价的原因吗？"，可能是真的成本上涨，也可能是试探。

### 招数 2：验证需求

"你涨价后我还能得到什么？"

对方说："我们要涨价 10%。"你问："涨价后我能得到额外的价值吗？"，如果不能，你应该考虑替代。

### 招数 3：引入替代

"涨价后我能不能考虑其他供应商？"

对方说："我们要涨价 10%。"你说："我可能需要看看市场上其他方案。"，给对方压力。

实战建议：

- 3 招组合用：先核查 + 验证，再决定是否引入替代。
- 不要立刻接受涨价，给对方压力。

---

## 八、报价的边界：哪些不能换 / 哪些可以换 / 哪些换时点

### 不能换的部分

不能换的核心是"原则性条款"，你的核心价值 / 你的法律底线 / 你的商业机密。

不能换：

- 你的核心商业模式。
- 你的核心知识产权。
- 涉及合规的条款（如数据安全）。

### 可以换的部分

可换的是"商业条款"，价格 / 时间 / 服务 / 范围。

可换：

- 价格（在底线之上）。
- 时间（在合理范围内）。
- 服务（在能力范围内）。
- 范围（在能力范围内）。

### 换时点

换的时间也有讲究：

- 谈判前期换大项（价格）。
- 谈判中期换中项（时间、服务）。
- 谈判后期换小项（细节）。

实战建议：

- 不能换的坚决不换，这是你的底线。
- 可以换的留到最后换，这是你的筹码。

## 九、一个正面案例：林涵的 360 万回血

（这一案例对应开篇引入的场景）

林涵那次失败后，她花了 1 个月专门研究报价策略。她和团队设计了 ERP 项目的三档报价。

第二次去见刘总时，她不再直接报价 380 万。她说：

"刘总，今天我准备了 3 个方案，您看看哪个适合您，"

- Good 档（280 万）：基础 ERP + 6 个月运维。
- Better 档（380 万）：完整 ERP + 12 个月运维 + 培训。
- Best 档（480 万）：完整 ERP + 24 个月运维 + 培训 + VIP 客户画像建模。

刘总看了 3 个方案，问了 30 分钟的问题。最后他说："我选 Better 档，但希望价格能到 360 万。"

林涵说："Better 档定价是 380 万。如果您选 Better 档，我可以维持 380 万，但加一项 3 个月的免费运维。如果您需要价格让步到 360 万，我希望您能签 3 年期合同 + 允许我们引用您作为案例。"

刘总想了 30 秒，同意。

林涵那次不仅没少赚，反而多赚了 30 万 + 一个长期合同 + 一个案例授权。

### 这个案例的关键：

- 多方案报价（3 档）。
- 不直接让价。
- 让步必换（让价换长期合同 + 案例授权）。

## 十、一个反面案例：周宁压价反被压

周宁做制造业采购，过去几年她的策略是"压价"，每次谈判都把价格压到最低。

她和一家供应商谈了 2 年。第 3 年供应商主动提价 15%。

周宁本能地开始压价："我们不接受，您们的成本不可能涨这么多。"

供应商说："周总，您过去 2 年已经把我们价格压到了成本线附近。今年材料涨价 18%，我们没办法。"

周宁继续压价 1 个月，最后供应商说："我们不能再降了，您可以考虑其他供应商。"

周宁花了 1 个月才找到新供应商，结果新供应商价格比原供应商高 25%，质量还不稳定。

她后来算账，过去 2 年她压价省下的钱，比这次重新找供应商多花的钱少 100 万。

### 这个案例的反面教训：

- 压价过头会导致供应商"反弹"，反弹后你损失更大。
- 总获取成本（TCO）比单价更重要。
- 长期合作比短期价格更重要。

## 十一、报价设计的 5 步流程

### 步骤 1：明确目标

你的理想值 / 可接受值 / 底线值 分别是多少。

### 步骤 2：设计 3 档

- Good 档（最低）= 你的底线 + 5%。
- Better 档（推荐）= 你的理想值 + 5%。
- Best 档（最高）= Better 档 × 1.3-1.5。

### 步骤 3：准备每档的具体内容

每档的价格 + 服务内容 + 时间 + 付款方式都要写清楚。

### 步骤 4：准备交换清单

列出你可以让步的 5 项 + 对方可以给你的 5 项。

### 步骤 5：演练

找一个朋友扮演客户，做 3 轮报价演练。演练时注意：

- 第 1 轮：报 3 档。
- 第 2 轮：应对压价。
- 第 3 轮：让步必换。

---

## 章末行动清单

下次报价 / 谈判前，提醒自己：

1. 我要先报 3 档（good-better-best）。
2. 我的第一报价要具体（精确到 1000 元）。
3. 我的让步必换（每次让步换 1 项）。
4. 我的让步幅度递减（10% → 7% → 5% → 3%）。
5. 我用量化价值应对压价（不要说"我们贵"，说"我们贵能换 X 价值"）。
6. 我不立刻接受涨价（先核查 + 验证 + 引入替代）。
7. 我准备 5 步报价设计流程。
8. 我不暴露底线（永远不要在开局时透露）。

---

## 实战演练

**演练 1**：为你手上的产品 / 服务设计三档报价。

- Good 档：基础功能 + 短期。
- Better 档：完整功能 + 中期。
- Best 档：完整功能 + 长期 + 加项。

设计后找 1 个朋友，让他选 1 档，记录他选的理由。

**演练 2**：复盘你最近一次失败的报价。用本节给出的 5 步流程重新设计，你会发现你可以拿回至少 30% 的损失。

**演练 3**：准备一份"交换清单"。列出你可以让步的 5 项 + 对方可以给你的 5 项。下次谈判时拿出来用。

## 本章小节

1. 高开低走适合有信息优势 / 强势地位；低开高走适合弱势地位 / 需要订单。
2. 第一报价要具体（精确数字）、基于事实、留 10-20% 余量。
3. 价格让步 4 铁律：不等量 / 不连续 / 不无理由 / 让步必换。
4. 多方案报价（good-better-best）让对方"选择"而不是"拒绝"。
5. 条件交换有 5 种：让价换让价 / 让价换时间 / 让价换规模 / 让价换案例 / 让价换关系。
6. 应对压价 4 招：量化价值 / 加项 / 切分 / 退场。
7. 应对涨价 3 招：核查成本 / 验证需求 / 引入替代。
8. 报价有"不能换"和"可以换"两类，不同时间换不同部分。
