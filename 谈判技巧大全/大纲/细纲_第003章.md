# 细纲：第 3 章 谈判准备

## 一句话核心事件

读者读完这一章，能在 30 分钟内完成"谈判准备清单"，并明确自己的 BATNA 是哪一项。

## 章首钩子（案例切入）

> "林涵接到一个 380 万的项目，她什么都没准备就跑过去谈，谈到第三轮对方说'我们找到一家 280 万也能做的'。林涵突然发现自己的 BATNA 是零。"

## 子情节点序列（约 12 个情节点）

1. BATNA 缺失的惨痛案例引入
2. 准备的 6 大块：信息 / 目标 / 底线 / BATNA / 风险 / 团队
3. 信息收集：3 类信息（对方底牌 / 自己手里 / 行业数据）
4. 对方背景分析：5 个固定维度（业绩 / 痛点 / 决策链 / 个性 / 时间偏好）
5. 明确自身目标：理想值 / 可接受值 / 底线 三档
6. BATNA 设计：3 个具体步骤（找替代 / 评价替代 / 隐藏 BATNA）
7. 风险评估：3 类风险（信息风险 / 决策风险 / 关系风险）
8. 准备一份简单的准备 Checklist（手写一份即可）
9. 一个"林涵 BATNA 复盘"案例：她后来如何把 BATNA 设计从 0 → 4 个
10. 准备的常见错误：临时抱佛脚 / 准备过头 / 把对方的预案当成你的预案
11. 准备的时间分配建议（短中长三种）
12. 章末行动清单 + 实战演练（用附录 A 的清单为下一次谈判做准备）

## 章首/章尾钩子

- 章首：没有 BATNA 的失败案例
- 章尾："今天/明天就要开会的谈判，你能在 30 分钟内完成这份准备吗？"

## 目标读者收获

1. 能列出 5 类准备的具体动作
2. 能写出自己的目标区间（理想值 / 可接受值 / 底线）
3. 能设计至少 1 个 BATNA
4. 能填一份风险评估表
